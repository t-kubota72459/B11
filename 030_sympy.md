## 2. 文字式（Symbol）の準備

数学の公式のように、文字を文字のまま扱うためには、まず「これは数学用の文字（シンボル）ですよ」と教える必要があります。いわゆる「◯◯を $x$ と置く」というやつです。

みなさんも中学校、高校で「$x^3 + 3x^2 + 3x +1$ の因数分解」という問題におびえた経験があると思います。もう恐れる必要はありません。

SymPy を使って、記号を含む数式を書いて実行すれば、わざわざ自分で計算しなくても、秒で答えが見つかります。

```python
# 従来の方法
x = 1
x + x + 1
```

上の式は $3$ という答が得られますが、記号 $x$ についての結果、$2x + 1$ がほしいときにはいままのでやり方、つまり **電卓機能のみ** ではできません。ここで SymPy の symbols() という機能をつかいます。

```Python
# 「x」という未知数の文字を定義する
x = sp.symbols('x')

# 文字式のまま計算し、きれいに表示（display）する
shiki = x + x + 1
display(shiki)

# もっと複雑な式も扱える
shiki = (x + 1) * (x - 1)
display(shiki)
```
> **💡 display() の魔法**  
> `print()` ではなく `display()` を使うと、数式が数学の教科書のようなきれいな見た目で表示されます。

### 例：オームの法則

  ```python
  I, R = sp.symbols('I R')
  V = I * R
  display(V)      # V = IR
  ```

## 3. 正確な計算：分数（Rational）

たとえば、エンジニアリングにおいて、1/7 を 0.142857... と小数で扱ってしまうと、計算が進むにつれて誤差が大きくなります。

### 例：2 メートルの鋼材から 1/7 の長さのパーツを切り出す

設計上、2m（2000mm）の鋼材から正確に 7 等分 したパーツを切り出したいとします。

1. &nbsp; &nbsp; **電卓で計算**: $2000 \div 7 = 285.714285...$[mm]
2. &nbsp; &nbsp; **メモを取る**: 「285.7mm」と小数点第１位で丸めてメモする。
3. &nbsp; &nbsp; **現場でしるし付け**: この「285.7mm」を基準に、端から順番に 7個分の印をつけていく。
4. &nbsp; &nbsp; **誤差が溜まっていく**:
   1. 本来の長さとの差（誤差）は、1個あたり約 **0.014mm** です。
   2. れを 7回繰り返すと：$0.014 \times 7 = 0.098$ mm
   3. さらに、これを100本、1000本と量産する場合、メモした数値（285.7）をそのまま使うと、数ミリ〜数センチの大きなズレとなり、設計寸法に収まらなくなります。
 
SymPyでは、分数を **正確な形のまま** 保持できます。

```python
# 普通の計算（小数になり、わずかな誤差が出る）
# 普通の電卓のうごき
print(1/7 + 1/7 + 1/7 + 1/7 + 1/7 + 1/7 + 1/7) 

# SymPyの分数（正確！）
# 分数のまま、保持してくれる
# sp.Rational(分子, 分母)
a = sp.Rational(1, 7)
display(a + a + a + a + a + a + a)  # 結果は正確に 1 になる
```

> **💡 エンジニアの知恵：**  
> 並列回路の合成抵抗計算など、逆数が登場する場面で `sp.Rational` は威力を発揮します。

## 4. 方程式を立てて解く（Eq と solve）

記号 $x$ の使い方や分数 Rational の使い方がわかったところで、次はいよいよ「方程式」を解いてみましょう。

**方程式とは**  
方程式（ほうていしき）とは、文字「$x$」など値が分からない未知数を含む「$=$（等号）」で結ばれた等式です。特定の文字の値（解）を代入した時だけ成り立つ式であり、その未知数の正体を見つけ出す「クイズ」のようなものです。

**例： $x + 5 = 12$ を解く**
  
$x$ がいくつなら 5 と足して 12 になるか？


### ⚠️ 注意：2つの「イコール」を使い分けよう

これからまったく違う意味の 2 つの「イコール」が出てきます。注意してください。

| 記号 | 意味 | 使い方 |
| :--- | :--- | :--- |
| **`=`** | **代入** | `x = 10` （x という箱に 10 を入れる。Pythonの機能） |
| **`sp.Eq()`** | **等式** | `sp.Eq(左辺, 右辺)` （数学の「＝」の関係を作る。左辺と右辺は等しい、とする SymPyの機能） |

**例： $x + 5 = 12$ を解く**

```python
import sympy as sp

# 1. 式を立てる： sp.Eq(左辺, 右辺)
# 左辺：x + 5 , 右辺：12
x = sp.symbols('x')
eq1 = sp.Eq(x + 5, 12)

# 2. 解く： sp.solve(式, 求めたい文字)
answer = sp.solve(eq1, x)

display(answer)  # 結果： [7] とリスト形式で表示されます
```

## 5. 【エンジニアの必須技能】公式の変形

現場のエンジニアは、状況に合わせて公式の形を書き換える必要があります。手計算だとミスしやすい「文字の入れ替え」も、Python なら一瞬です。

**課題：オームの法則 $V = I \times R$ を「$I$ について解く（ $I = \dots$ の形にする）」**

```python
# 必要な文字をまとめて定義
v, i, r = sp.symbols('v i r')

# 公式を定義： V = I * R
ohm_siki = sp.Eq(v, i * r)

# i について解く（第2引数に「解きたい文字」を指定する）
i_siki = sp.solve(ohm_siki, i)

print("電流 I を求める式は：")
display(i_siki) 
```

> **💡 なぜこれをするの？**  
> 公式を一つ（$V=IR$）だけ定義しておけば、あとは Python が勝手に $I$ についても $R$ についても解いてくれます。たくさんの公式を暗記する必要がなくなるのです。

## 6. 値を当てはめて計算する（subs と evalf）

最後に、導き出した公式に具体的な数値を代入して、現場で使える「値」を出してみましょう。

**i_siki[0]** ： 答が 2つ以上、複数ある場合があるので、その 0 番目を取り出す、という意味。

```python
# 1. 代入（subs）： 先ほどの I の式に、v=10, r=3 を入れる
# i_siki[0] は「答えのリストの1番目」を取り出す書き方です
# V=10V, R=3Ω
ans_fraction = i_siki[0].subs({v: 10, r: 3})

print("分数のままの答え：")
display(ans_fraction)  # 結果： 10/3 （正確！）

# 2. 数値化（evalf）： 現場で使いやすい「小数」に直す
ans_value = ans_fraction.evalf()

print("小数（エンジニア表示）の答え：")
display(ans_value)
```

## 7. 実習：並列回路の合成抵抗を求めよう

第2回で学んだ「e表記」と、今回の「分数」「公式変形」を組み合わせてみましょう。

**問題： $R_1 = 1.0\text{k}\Omega$ ($1e3$), $R_2 = 2.2\text{k}\Omega$ ($2.2e3$) の並列合成抵抗 $R$ を求めよ。**

公式： $\frac{1}{R} = \frac{1}{R_1} + \frac{1}{R_2}$

```python
# 1. 文字を定義
R, R1, R2 = sp.symbols('R R1 R2')

# 2. 公式を立てる
teikou_heiretu = sp.Eq(1/R, 1/R1 + 1/R2)

# 3. R について解く
r_siki = sp.solve(teikou_heiretu, R)

# 4. 数値を代入して計算（R1=1K, R2=2.2K）
final_r = r_siki[0].subs({R1: 1e3, R2: 2.2e3})

print("合成抵抗 R [Ω] は：")
display(final_r.evalf())
```
---
### まとめ

1. **立式**： `sp.Eq()` で現象を数式にする。
2. **変形**： `sp.solve()` で欲しい文字を左辺に持ってくる。
3. **代入**： `subs()` で数値を入れ、 `evalf()` で答えを出す。

この **「立式 → 変形 → 代入」** の流れこそが、コンピュータを使いこなすエンジニアの標準ワークフローです。

## 8. 注意点：SymPy でよくやるミス
| 誤った書き方 | 理由 | 正しい表記 |
| :--- | :--- | :--- |
| `x + 1 = 0` | `=` は代入なので方程式にならない | `sp.Eq(x + 1, 0)` |
| `2x` | 文字と数字の間の `*` を省略できない | `2 * x` |
| `x^2` | `^` は数学用ではない | `x ** 2` |
| `solve(x+1)` | 右辺を省略すると自動的に `= 0` とみなされる | `sp.Eq(x+1, 0)` と書くのが安全 |
