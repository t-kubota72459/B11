## 1. なぜエンジニアは 2進数 と 16進数 を使うのか？

皆さんが将来扱うロボット、NC 工作機械、自動車のエンジン制御ユニット（ECU）の中には、必ず「マイコン（小さなコンピュータ）」が入っています。

* **2進数（Binary: バイナリ）**  
  マイコンの脳内は、スイッチの「ON（電気が通っている）」と「OFF（電気が通っていない）」しか分かりません。これを「1」と「0」の2種類の数字だけで表したのが2進数です。

* **16進数（Hexadecimal: ヘックス）**  
  しかし、2進数は `10101111` のように人間にとって長すぎて読みにくく、現場での入力ミス（ヒューマンエラー）の原因になります。
  そこで、**2進数を4桁ずつまとめて短く表現できる「16進数」** （0〜9とA〜Fを使う）が、人間と機械の間の「共通言語」として現場で使われます。（例：モーターの制御パラメータ設定や、ネットワークのIPアドレスなど）

## 2. Python で N進数を変換してみよう

Python の組み込み関数を使って、私たちが普段使う10進数を、機械の言葉（2進数や16進数）に一瞬で変換してみましょう。

* `bin()` : 2進数に変換。頭に **`0b`**（ゼロ・ビー：バイナリの略）が付きます。
* `hex()` : 16進数に変換。頭に **`0x`**（ゼロ・エックス：ヘックスの略）が付きます。

```python
# 人間の数字（10進数）
num = 255

print("2進数:", bin(num))  # 結果: 0b11111111
print("16進数:", hex(num)) # 結果: 0xff
```

> **💡 エンジニア・メモ： 関数電卓での N進法計算**  
> 皆さんの関数電卓にも必ず「N進計算モード（BIN / OCT / HEX / DEC 等）」がついています。
> Pythonの `bin()` や `hex()` と同じように、手元で瞬時に変換できます。
> 機械系のエンジニアは、現場で PLC（シーケンサ）のメモリ番地を計算したり、センサーの値を読み解いたりするときに、この電卓の機能を毎日使い倒します。

## 3. 「集合のAND/OR」＝「機械のビット演算」

前回、ベン図を使って要素の「AND（`&`：積集合）」や「OR（`|`：和集合）」をやりました。
機械の世界では、これを **「1（ON）」** と **「0（OFF）」** の計算として行います。これを **ビット演算** と呼びます。

Pythonでは、対象が「集合」ならベン図の計算になり、「数字」ならビット演算になります。（使う記号は全く同じです！）

### ① ビットごとの AND（`&`）

両方の同じ桁が `1` のときだけ `1` になります。前回の **「cap（キャップ：フタをして限定する）」** の動きです。

```python
a = 0b1100  # 10進数だと 12
b = 0b1010  # 10進数だと 10

# AND演算（両方 1 の桁だけが 1 に残る）
ans_and = a & b
print("ANDの結果:", bin(ans_and)) # 0b1000 になる
```

### ② ビットごとの OR（`|`）

両方の同じ桁のどちらか一方が `1` なら `1` になります。前回の **「cup（カップ：すくい取る）」** の動きです。

```python
# OR演算（どちらかが 1 なら 1 としてすくい取る）
ans_or = a | b
print("ORの結果:", bin(ans_or)) # 0b1110 になる
```

## 4. エンジニアの実践：ビットフラグ（フラグ管理）

なぜわざわざ 0 と 1 の計算をするのか？
それは **「複数のセンサーのON/OFF状態を、たった1つの数字でまとめて管理する」** ためです。

例えば、工場にあるロボットアームに4つのセンサーが付いているとします。
（各ビットが、左から：①温度異常, ②圧力異常, ③モーター動作中, ④電源ON を表す）

* 状態が `0b1001` なら ＝ ①温度異常(1) かつ ④電源ON(1)。他は正常(0)。

**【課題：特定のセンサーの状態だけを調べる（ANDマスク）】**  

「③モーターが動いているか（右から2番目のビットが1か）」だけをプログラムで調べたい時、エンジニアは **調べたい場所だけを `1` にした「マスク」** を作って、AND（`&`）演算で他の情報をフタ（cap）してしまいます。

```python
robot_status = 0b1011  # 現在のロボットの状態（温度異常, モーター稼働, 電源ON）

# 調べたい場所（右から2番目のモーター）だけを 1 にしたマスクを作る
check_motor = 0b0010

# AND演算で抽出する（フタをして、モーターの桁だけを見る）
result = robot_status & check_motor

print("モーターの確認結果:", bin(result))
# 結果が 0b0 なら止まっている、0b1 なら動いていると判定できる
```

このように、不要な情報を「0」で潰して、欲しい情報だけを取り出すテクニックを「**ビットマスク**」と呼びます。機械制御のプログラミングでは絶対に必要なテクニックです。

### 練習2：工作機械の異常検知（ビットマスク）

ある NC 工作機械は、8つのセンサー（8ビット）で状態を管理しています。

現在の機械の状態は `status = 0b11010100` です。

左から3番目のビット（非常停止ボタンが押されているか）だけを確認したいとします。
以下のコードを完成させて、非常停止ボタンの状態を抽出してください。

```python
status = 0b11010100  # 現在の機械の状態

# 1. 調べたい場所（左から3番目）だけを 1 にしたマスクを作る
# (ヒント：8桁の2進数で、左から3番目だけを1にする)
mask = 0b00100000

# 2. status と mask を AND演算(&) して結果を表示する
check = status & mask
print("非常停止の確認結果:", bin(check))
```
